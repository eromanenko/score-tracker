# Інструкція з налаштування ігор (Score Tracker)

Кожна гра у додатку використовує певний тип підрахунку балів (`scoringType`), який визначається у файлі `assets/games.json`. Детальні правила поведінки гри та відображення елементів задаються у конфігураційному файлі `config.json` всередині ZIP-архіву гри (наприклад, `assets/games/star-realms.zip`).

Нижче наведено перелік усіх доступних типів підрахунку та їхніх параметрів конфігурації (`config.json`).

---

## 1. Тип: `category` (Категорійний підрахунок)
**Де використовується:** Ігри, де в кінці партії потрібно підрахувати бали за різними категоріями (наприклад, "7 Чудес", "Castle Combo"). Переможець завжди визначається за найбільшою кількістю балів.

**Налаштування `config.json`:**
- `headerIconFile` *(String, опціонально)*: Назва файлу іконки в архіві для заголовка стовпця "Категорії".
- `totalIconFile` *(String, опціонально)*: Назва файлу іконки в архіві для рядка "Разом".
- `categories` *(Array of Objects)*: Масив об'єктів, кожен з яких описує одну категорію для підрахунку.
  - `id` *(String)*: Унікальний ідентифікатор категорії (наприклад, `"military"`).
  - `nameEN`, `nameUK`, `nameRU` *(String)*: Локалізовані назви категорії, які відображатимуться в інтерфейсі (наприклад, `"Військові бали"`).
  - `hintEN`, `hintUK`, `hintRU` *(String, опціонально)*: Локалізовані підказки. Якщо вказано, при натисканні на назву категорії з'явиться спливаюче повідомлення (Toast) із поясненням правил підрахунку для цієї категорії.
  - `color` *(String, опціонально)*: CSS-колір, який використовується для підсвічування поля вводу та тексту цієї категорії (наприклад, `"#ef4444"` для червоного).
  - `divider` *(Number, опціонально)*: Дільник. Якщо вказано, введене значення буде поділено на це число (націло). Наприклад, у "7 Чудес" 3 монети дають 1 бал, тому `divider: 3`.
  - `icon` *(String, опціонально)*: Текстовий емодзі для категорії (наприклад, ⚔️).
  - `iconFile` *(String, опціонально)*: Назва файлу іконки (наприклад, `"military.svg"`), який лежить всередині ZIP-архіву разом з `config.json`.

---

## 2. Тип: `cumulative` (Накопичувальний підрахунок)
**Де використовується:** Ігри, що складаються з кількох раундів, де бали накопичуються після кожного раунду. Гра закінчується, коли хтось досягає певного ліміту (наприклад, "Корова 006", "Flip 7").

**Налаштування `config.json`:**
- `targetScore` *(Number)*: Цільовий рахунок. Гра автоматично закінчується наприкінці раунду, коли хоча б один гравець досягає або перетинає цю позначку.
- `winCondition` *(String)*: Умова перемоги, визначає, хто виграв після досягнення `targetScore`. Можливі значення:
  - `"highest"`: Перемагає гравець із найбільшою кількістю балів (наприклад, "Flip 7").
  - `"lowest"`: Перемагає гравець із найменшою кількістю балів (штрафних). Використовується, наприклад, у грі "Корова 006".

---

## 3. Тип: `tracker` (Трекер реального часу)
**Де використовується:** Ігри, де гравці додають або віднімають бали в режимі реального часу протягом усієї партії (наприклад, "Зоряні імперії", де відслідковується рівень здоров'я).

**Налаштування `config.json`:**
- `startScore` *(Number)*: Початковий рахунок (або кількість "життів"), який видається кожному гравцю на початку гри (наприклад, `50`). Якщо не вказано, за замовчуванням рахунок 0.
- `buttons` *(Array of Numbers)*: Масив чисел, які відображатимуться як швидкі кнопки навколо рахунку гравця.
  - Додатні числа (наприклад, `5`, `10`) створять кнопки `+5`, `+10` у верхній зоні.
  - Від'ємні числа (наприклад, `-5`, `-10`) створять кнопки `-5`, `-10` у нижній зоні.
- `minScore` *(Number, опціонально)*: Нижня межа рахунку. Рахунок гравця ніколи не опуститься нижче цього значення. Для "Зоряних імперій" зазвичай це `0`.
- `maxScore` *(Number, опціонально)*: Верхня межа рахунку. Рахунок гравця ніколи не перевищить це значення.
- `winCondition` *(String, опціонально)*: Умова закінчення гри. Наразі доступно:
  - `"last_standing"`: Гра автоматично оголошує переможця, коли лише один гравець залишається "живим" (його рахунок строго більший за `minScore` або `0`). Якщо всі залишені гравці одночасно втрачають "життя", оголошується нічия ("Draw").
  - `"first_to_max"`: Гра оголошує переможця, коли гравець досягає або перевищує значення `maxScore` (наприклад, у грі "Dnup").

---

## Загальні параметри (реєстрація гри в `games.json`)
Щоб гра з'явилася у головному меню, її потрібно додати до масиву `games` у файлі `assets/games.json`. 

Об'єкт гри має такі параметри:
- `id` *(String)*: Унікальний ідентифікатор гри (наприклад, `"7-wonders"`).
- `nameEN`, `nameUK`, `nameRU` *(String)*: Локалізована назва гри, що відображається у списку меню та в заголовку трекера.
- `minPlayers`, `maxPlayers` *(Number)*: Ліміти кількості гравців, які можна обрати на екрані налаштування партії (наприклад, `minPlayers: 2`, `maxPlayers: 8`).
- `scoringType` *(String)*: Вказує, який тип компоненту використовувати для цієї гри. Допустимі значення: `"category"`, `"cumulative"`, `"tracker"`.
- `iconUrl` *(String)*: Емодзі (наприклад, `"🃏"`) або відносне посилання на картинку-іконку (наприклад, `"./assets/icons/game.png"`), що показується в меню.
- `bundleUrl` *(String)*: Відносний шлях до ZIP-архіву, який містить `config.json` та всі додаткові ресурси (картинки, іконки) для цієї конкретної гри (наприклад, `"./assets/games/flip-7.zip"`).
